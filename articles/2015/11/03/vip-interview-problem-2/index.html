解释面向对象语言的特性

Java三大面向对象特性：封装，继承，多态。

  • 封装：隐藏对象的属性和实现细节，仅对外公开接口,控制在程序中属性的读和修改的访问级别。
  • 继承：通过继承实现代码复用。继承可以使得子对象具有父对象的各种属性和方法，而不需要再次编写相同的代码。子类可以复用父类的方法和属性，也可以自己再定义。
  • 多态：同一行为的多种不同表达，或者同一行为的多种不同实现就叫做多态。

多线程中为什么要进行同步操作？

线程的同步是指让多个运行的线程在一起良好地协作，达到让多线程按要求合理地占用释放资源。我们采用Java中的同步代码块和同步方法达到这样的目的。 数据同步就是指在同一时间，只能由一个线程来访问被同步的类变量，当前线程访问完这些变量后，其他线程才能继续访问。

描述数组和链表的区别

  • 数组：元素在内存中连续存放，根据索引访问元素速度特别快，插入或删除元素速度慢。
  • 链表：元素在内存中不是顺序存储的，而是通过存在元素中的指针联系到一起。访问链表中的元素，必须从头依次遍历，速度很慢，反而插入或删除元素速度很快，只需修改几次指针的指向。

如何判断链表是否有环

/**
 * 判断链表是否带环，我们可以采用在头结点设两个指针，一个叫fast，一个叫slow，
 * fast一下走两步，而slow一下走一步。
 * 如果链表中存在环的话，那么fast和slow必定会在环中相遇。
 * 若链表中没有环的话，那么fast必定现于slow指针先到达链表的尾节点（->next = Null）。
 */
public boolean hasRing(Node head) {
    Node fast = head;
    Node slow = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;//步长为1
        fast = fast.next.next;//步长为2

        if (slow == fast) return true;
    }

    return false;
}

描述红黑树的实现思路

二三树中有两种节点：

  • 二节点对应一个键，有两个子节点
  • 三节点对应两个键，有三个子节点

二三查找树非常平衡，每个空节点到根节点的距离都是一样的 。

红黑树就是将二三树表示成二叉树的形式，极大地简化了算法。

红黑树的基本思想就是将二三树中的三节点表示成两个二节点，而这两个二节点之间使用红色的连接，普通连接使用黑色的连接。

红黑树中的每个节点都有以下性质：

  • 没有一个节点同时拥有两个红连接
  • 每个空节点到根节点路径上黑色连接的数量都是相同的
  • 红连接只会出现在左边

描述快速排序的思路，并写出代码

基本思想

  1. 先对输入的数据进行洗牌
  2. 以数据a[j]为中心进行分区，使得a[j]左侧的数据都小于等于a[j]，a[j]右侧的数据都大于等于a[j]
  3. 分区完后递归排序
/**
 * Created by zero on 2015/11/03.
 * QuickSortDemo
 */
public class Quick {

    public static void sort(int[] a) {
        //为了保障快速排序的性能, 在排序之前进行打乱操作
        shuffle(a);
        sort(a, 0, a.length - 1);
    }

    private static void sort(int[] a, int lo, int hi) {
        if (hi <= lo) return;

        //切分
        int i = partition(a, lo, hi);

        //左子数组排序
        sort(a, lo, i - 1);
        //右子数组排序
        sort(a, i + 1, hi);
    }

    private static int partition(int[] a, int lo, int hi) {
        int i = lo, j = hi + 1;

        while (true) {
            //从左扫描出第一个大于a[lo]的元素
            while (a[++i] < a[lo]) if (i == hi) break;
            //从右扫描出第一个小于a[lo]的元素
            while (a[--j] > a[lo]) if (j == lo) break;

            //上述元素不存在
            if (i >= j) break;
            //将较小元素移至a[lo]左边,较大元素移至右边
            swap(a, i, j);
        }

        //将a[lo]移至中间
        swap(a, lo, j);
        return j;
    }

    private static void swap(int[] a, int i, int j) {
        int t = a[i];
        a[i] = a[j];
        a[j] = t;
    }

    private static void shuffle(int[] a) {
        Random random = new Random();
        for (int i = 0; i < a.length; i++) {
            int j = random.nextInt(a.length);
            swap(a, i, j);
        }
    }
}

Android中为什么要使用多线程？

Android中主线程即UI线程，主要绘制UI界面。 进行一些耗时操作时，如I/O读写大文件、数据库操作、访问网络及下载任务等，需要很长时间，如果在主线程进行这些操作，会阻塞用户界面，从而导致出现ANR(Application Not Response)失去响应提示框，进而程序挂掉。 因此需要使用多线程，将耗时操作放在子线程后台完成，然后通过Handler等异步处理进行数据传递。
